## ğŸ¤ How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issues tab
- Include a clear title and description
- Provide steps to reproduce the bug
- Include screenshots if applicable
- Mention your browser/device

### Suggesting Features
- Open an issue with the tag "enhancement"
- Clearly describe the feature and its benefits
- Explain how it fits into the cyberpunk theme

### Pull Requests
1. Fork the repository
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages: `git commit -m "Add: new feature description"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Open a Pull Request

## ğŸ“ Code Style Guidelines

### TypeScript
- Use TypeScript for all new files
- Define proper interfaces for data structures
- Avoid `any` types when possible

### React
- Use functional components with hooks
- Keep components small and focused
- Use meaningful component and variable names

### Styling
- Use Tailwind CSS classes
- Maintain the cyberpunk aesthetic (#0A84FF theme)
- Ensure responsive design (mobile + desktop)
- Keep glow effects and animations consistent

### Naming Conventions
- Components: PascalCase (e.g., `TaskList`)
- Functions: camelCase (e.g., `handleTaskComplete`)
- Constants: UPPER_SNAKE_CASE (e.g., `API_BASE`)
- Files: kebab-case (e.g., `task-list.tsx`)

When adding features, keep these in mind:
- **Cyberpunk Aesthetic**: Neon blue (#0A84FF), glowing effects, grid patterns
- **Performance**: Smooth animations, no lag
- **Responsiveness**: Works on all screen sizes
- **Simplicity**: Clean, intuitive UI
- **Gamification**: Make progress tracking fun and engaging
